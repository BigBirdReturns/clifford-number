# Methodology

The Clifford Number is a shortest-path topology tool. It maps evidenced institutional adjacency to a target node called the Clifford Policy Machine. The current seed graph uses UK AI policy as the target field and Dialog as one public social-layer case.

The method is intentionally dull. Every edge has a type, a claim, an evidence class, a status, and at least one source. The joke is allowed to be sharp only if the underlying edge is boring enough to audit.

## What the number means

A Clifford Number of `1` means the node has a direct sourced edge to the target. A number of `2` means there is one intermediary node. The number is not a moral score, guilt score, corruption score, ideology score, or intent score.

The number only says this:

```txt
Under the current public graph, this is the shortest evidenced institutional path to the target node.
```

## Target node

The target node is `clifford-policy-machine`. It represents the policy machine around Matt Clifford, the AI Opportunities Action Plan, the Prime Minister adviser role, public-sector AI adoption, national AI infrastructure, government as AI customer, and state-market formation.

The target can be changed in `graph.json` by editing `target_node_id`.

## Evidence classes

`confirmed` is direct official sourcing or tier-one reporting.

`reported` is credible reporting or public profile sourcing.

`derived` is an analytic edge where the facts are sourced but the frame is inferred.

`judgment` is an interpretive label used to help navigate the graph.

`open` is not publication-ready.

## Status separation

The app keeps `listed`, `registered`, and `attended` separate. This is non-negotiable. A name on a list does not prove attendance. Registration does not prove attendance. Attendance does not prove agreement. None of those prove wrongdoing.

## Why topology

A scandal frame asks what someone secretly intended. A topology frame asks what the structure allows, what cannot be audited, what cannot be removed, and what public lever exists against it.

This repo should stay inside the topology frame.
